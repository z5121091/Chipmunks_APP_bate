# 掌上仓库同步助手说明

`label_sync_server.py` 是掌上仓库 APP 的电脑端同步助手。手机端把 Excel 数据发送到本机服务后，同步助手会把文件保存到电脑目录，便于后续打印、归档或导入其他系统。

## 运行方式

### 直接运行

```bash
pip install cherrypy openpyxl pystray Pillow pyinstaller
python label_sync_server.py
```

### 打包成 EXE

推荐使用仓库中的打包脚本：

```bat
build.bat
```

打包后双击 `dist\掌上仓库同步服务.exe` 即可运行。运行后程序会常驻系统托盘。

## 服务地址

默认端口：`8080`

健康检查地址：

```text
http://电脑IP:8080/health
```

手机端在“设置 -> 电脑同步”里填写电脑 IP 和端口后，可以测试连接。

## 数据保存目录

默认保存目录：

```text
D:\数据同步
```

同步助手会自动创建目录。手机端同步成功后，会返回实际保存的文件名。

## 当前接口

| 接口 | 用途 |
| --- | --- |
| `/health` | 检查同步助手是否在线 |
| `/inbound` | 接收入库单 Excel |
| `/outbound` | 接收出库单 Excel |
| `/inventory` | 接收盘点单或盘点拆包标签 Excel |
| `/labels` | 接收订单拆包标签 Excel |
| `/materials` | 接收物料绑定 Excel |

## 文件命名规则

同步助手支持两种命名方式：

- `file_name`：手机端传入精确文件名时，电脑端按该名称保存。
- `name_suffix`：手机端只传入后缀时，电脑端按业务类型拼接文件名。

例如：

```text
入库单_无锡总仓_20260518_02.xlsx
出库单_无锡总仓_20260518_01.xlsx
盘点单_无锡总仓_完整数据_20260518_01.xlsx
盘点拆包标签_20260518.xlsx
订单标签_20260518.xlsx
```

如果手机端传入 `file_name`，以手机端文件名为准；这是当前 APP 重新同步历史单据时使用的主要方式。

## Excel 处理说明

- 同步助手接收的是手机端生成的 `.xlsx` 文件。
- 入库、盘点等多 Sheet 文件会保留原 Sheet 结构。
- 同步助手会补充基础字体、边框、居中和列宽。
- 为避免历史数据很大时卡顿，样式处理只采样前面部分行。

## 周次转换工具

托盘工具中包含芯片周次转换功能，支持常见周次写法，例如：

```text
2601
2602S
202601
2026-W01
```

转换结果以周一为准，适合把芯片周次换算为生产日期。

## 常见问题

### 手机连接失败

1. 确认电脑和手机在同一局域网。
2. 确认同步助手正在运行，托盘图标没有退出。
3. 在电脑浏览器打开 `http://localhost:8080/health`，确认服务正常。
4. 检查 Windows 防火墙是否放行 8080 端口。

### 同步失败但服务在线

1. 检查 `D:\数据同步` 是否有写入权限。
2. 确认目标 Excel 文件没有被 WPS/Excel 正在占用。
3. 查看同步助手同目录下的 `日志记录.log`。
